# Public Neutrality

This repository is public-first and example-driven.

All docs, tests, examples, fixtures, package names, comments, and exported APIs must use synthetic, generic concepts. They must not reveal or imply organization-specific, product-specific, deployment-specific, private runtime, or proprietary workflow details.

## Allowed Examples

- A tiny retrieval example using invented documents.
- A synthetic support workflow with generic labels.
- A toy tool-calling loop with made-up inputs and outputs.
- In-memory or local-only observability backends.

## Review Checklist

- Examples are synthetic and reusable by any Python user.
- Identifiers are generic and descriptive.
- No private paths, network addresses, payloads, or operational assumptions are present.
- Public APIs describe observability concepts rather than one deployment.
- Release notes, article companion pages, and package metadata do not include
  unpublished repository URLs or private workspace paths.
