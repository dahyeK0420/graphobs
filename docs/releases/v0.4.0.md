# 0.4.0

`0.4.0` is a complexity-reduction pass. It removes unused generality — including
the experimental discovery subsystem and the trace payload-serializer override —
flattens the contract execution lifecycle, and folds single-purpose modules into
their shared homes, with no change to the package-root interface.

## Highlights

- `ProjectionPolicy` is now include-only. The `exclude` and `summarize` axes are
  removed; they were never set outside tests, and span payloads are already kept
  compact by the default trace serializer. Declare boundaries as dotted include
  paths.
- The overlapping contract and policy protocols are consolidated. The redundant
  `ContractProjection` and `ProjectionPolicyLike` protocols are removed from
  `graphobs.contracts.projection`, leaving `Contract` (the node/subgraph shape)
  and `PathPolicy` (the include-only policy shape).
- The experimental contract-discovery subsystem (`graphobs.discovery`) is
  removed. It had no runtime, package-root, or example caller. Design node
  contracts directly, and use `AUDIT` mode to surface a node's real boundary.
- The `graphobs.tracing` payload-serializer override is removed
  (`PayloadSerializer`, `default_payload_serializer`, and the `serializer=`
  parameter on the span helpers); no caller overrode it. Select payload shape
  with `mode=` / `TracePayloadMode`.
- The contract execution lifecycle is flattened: the `ContractRunSpec` dataclass
  and the two spec factories are replaced by a single parameterized
  `instrument_contract_run` / `instrument_contract_arun` pair.
- Single-purpose modules are folded into shared homes: `contracts/validation.py`
  and `langgraph/read_audit.py` move into `contracts/conformance.py`, and the
  payload policy from `_observability/payload_policy.py` merges into
  `payloads.py`. The `_observability` and empty `_integrations` packages are
  removed.

## Migration Notes

The package-root interface is unchanged. Code that imports lower-level submodules
should note:

- `ProjectionPolicy(..., exclude=..., summarize=...)` no longer accepts those
  keywords. Use `ProjectionPolicy(include=...)` or a plain tuple of dotted paths,
  and rely on the default trace serializer to keep heavy values compact.
- `graphobs.contracts.projection.ContractProjection` and `ProjectionPolicyLike`
  are removed; use `graphobs.contracts.models.Contract`. `delete_path` is removed
  from `graphobs.state.paths`.
- `graphobs.discovery` is removed. Replace `discover_contract` /
  `assert_contract_matches` usage with hand-written contracts plus `AUDIT` mode.
- `graphobs.contracts.validation.validate_update` and
  `graphobs.langgraph.read_audit.enforce_undeclared_reads` now live in
  `graphobs.contracts.conformance`. The payload helpers move from
  `graphobs._observability.payload_policy` to `graphobs.payloads`.
- The `serializer=` override and `PayloadSerializer` /
  `default_payload_serializer` are removed from `graphobs.tracing`; use
  `mode=TracePayloadMode.*`.
- `LogContext.as_attributes` is removed; use `as_metadata()`.
- `ContractRunSpec`, `node_contract_run_spec`, and `subgraph_contract_run_spec`
  are removed from `graphobs.langgraph.execution`; the node and subgraph wrappers
  now call `instrument_contract_run` / `instrument_contract_arun` directly.
